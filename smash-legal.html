3 Data in detail

3.1 On your device

  • SwiftData: History entries (structured reply/analysis data, short preview text, optional compressed screenshot of the chat view).
  • SwiftData: Preferences (e.g. style, goals, profile hints for suggestions).
  • UserDefaults: local app state (onboarding, tab selection, free-use counters, streaks — not a cross-app profile).
  • Keychain: session tokens for backend sign-in when you use Sign in with Apple.

3.2 Photos

You pick images with the system photo picker. Only the pixels you select are processed to generate suggestions or analysis. The App does not scan your whole photo library in the background.

3.3 AI (text + optional screenshot)

Encryption in transit: Requests travel only over HTTPS (TLS) between the App and our servers, which protects data while it crosses the internet using standard transport encryption.

No direct client-to-model call: The App does not call the AI provider directly. Flow:

  • The App sends an HTTPS request to our Supabase Edge Function grok-proxy (/functions/v1/grok-proxy).
  • Only that function forwards content to xAI's Grok models (e.g. grok-4-1-fast-non-reasoning) and returns the result.
  • This keeps API keys and abuse controls on our server side, not inside the App.

Content: Text and/or image data as needed; screenshots may be encoded (e.g. base64 in the request). Purpose: reply suggestions or chat analysis.

Third parties in chats: content may include other people's personal data. You must only submit what you are allowed to use.

3.4 Account & backend (Supabase) — optional

You may use features with or without a long-lived account. Supabase Inc. provides auth, database, and Edge Functions for us.

  • Sign in with Apple: Apple provides an identifier and optionally an email (per your Apple choices).
  • Without Apple: an anonymous Supabase account may be used (device-bound; see in-App flows).
  • Table smash_users stores e.g. internal user UUID, optional Apple linkage, email field, timestamps; server-side fields may include Pro status as implemented.
  • Table smash_history may hold history that the App syncs to the server only when you are signed in (goal/platform, JSON for replies/analysis, preview text, optional base64 screenshot). Without the relevant session/account, that server sync does not run as described.
  • Other Edge Functions (e.g. the AI proxy grok-proxy) run in the same Supabase project.

3.5 Subscriptions (Smash Pro)

Purchases run only through the Apple App Store (StoreKit). Apple processes payment data. Entitlement is evaluated from Apple transactions on the device.
